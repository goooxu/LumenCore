# 02 渲染方程入门

## 先建立直觉

你看见一个红色塑料球，其实是：

1. 光源发出光；
2. 光打到球表面，一部分被吸收，一部分被反射到各方向；
3. 其中一小部分刚好射进相机（或你的眼睛）。

路径追踪要回答的问题是：**对屏幕上每个像素，最终有多少「亮度」从那个方向射进来？**

![半球与入射/出射](figures/hemisphere-rendering.avif)

图注：在表面一点，法线 $N$ 把空间分成「朝上」的半球。出射方向上的光 $`L_o`$，来自半球里各个入射方向 $`L_i`$ 的贡献。

## 渲染方程（最简形式）

在表面点 $x$、出射方向 $`\omega_o`$ 上，出射辐射亮度可以写成：

```math
L_o(x,\omega_o)=L_e(x,\omega_o)+\int_{\mathcal{H}^+} f_r(x,\omega_i,\omega_o)\,L_i(x,\omega_i)\,(\omega_i\cdot n)\,d\omega_i
```

用中文拆开：

| 符号 | 含义 |
|------|------|
| $`L_o`$ | 沿 $`\omega_o`$ **出去**的光 |
| $`L_e`$ | 表面自己**发光**（灯、屏幕） |
| $`f_r`$ | **BSDF / BRDF**：材质如何把入射光反射到出射方向 |
| $`L_i`$ | 沿 $`\omega_i`$ **射进来**的光 |
| $`(\omega_i\cdot n)`$ | 斜射变弱的几何因子（余弦） |
| $`\int_{\mathcal{H}^+}`$ | 对法线朝上的**整个半球**积分 |

直觉口诀：

> **出射 = 自发光 + 「各方向入射 × 材质 × 余弦」的总和。**

## 为什么难？

积分里的 $`L_i`$ 往往又等于别的表面上的 $`L_o`$（间接光、镜面反射、透过玻璃再反射……）。于是方程自己套自己，形成**递推**。

路径追踪用随机采样把这条递推展开成一条条「路径」，再平均——这就是下一章的蒙特卡洛。

## 和光栅化着色的关系

Phong / Blinn 等实时模型，可以看成对上面积分的**极粗近似**（只取几个灯方向）。路径追踪则尽量「老老实实」估这个积分，所以能自然得到：

- 软阴影
- 颜色渗透（color bleeding）
- 镜面反射里映出别的物体
- 玻璃折射后的焦散（足够样本时）

![Cornell Box](../../outputs/cornell.avif)

图注：康奈尔盒。红墙绿墙的间接光照到白盒子上，就是积分里「别的表面反射来的 $`L_i`$」。

## 本项目里方程落在哪？

在 `src/device/shaders.cu` 的 closesthit 中：

1. 若材质有 `emission`，先加到 `prd->radiance`（对应 $`L_e`$）。
2. 用不透明 BSDF（`eval_opaque_bsdf` / `sample_opaque_bsdf`）或玻璃路径，估计反射/折射项。
3. 用 NEE / HDRI 额外估计来自灯与环境的 $`L_i`$。

材质细节见 [04](04-materials-bsdf.md)；直接连灯见 [05](05-nee-mis-hdri.md)。

## 小结

- 渲染方程描述「这点往某方向看出去有多亮」。
- 难在半球积分 + $`L_i`$ 的递归。
- 路径追踪 = 用随机路径去估这个积分。
